V-Mock

概述
  1. 欢迎你使用V-Mock，如需其他用户注册，可以去系统参数中修改【邀请码】, 普通用户即可通过邀请码注册系统
  2. 别忘了修改管理员的密码
  3. 登录账号不能修改，可以修改系统中显示的名称
  4. 忘记密码请联系管理员协助【重置密码】，默认为123456
  5. 响应格式可以在【自定义响应头】增加content-type自己指定, 默认content-type为application/json;charset=utf-8
  6. 注意！不要配置矛盾的URL，如：配置了/v1/test/{path}又配置了/v1/test/test
接口一览

接口一览是主要配置模块，使用系统可以遵循以下的步骤。ps: 以下文字中，中括号【】为强调，配置时请忽略。

  1. 在接口一览中，点击【添加】，新建一个需要模拟的URL，如果有path传参，请一定用【{path}】进行占位。如/v1/{path}/test/{path}
  2. 添加一个URL后，请继续添加该URL的返回体，点击URL后方的【返回体】按钮，就可以进入该URL的返回体列表。
  3. 返回体可以指定自定义响应码(Http Code)，但尽量不要乱写，例如写个789，部分浏览器可能对该请求无响应。
  4. 返回体列表中，状态一栏的按钮可以点击，用来切换当前使用返回体。
  5. 按照您配置的请求信息，进行请求，即可看到效果，请求成功后，系统会记录详细日志(【接口日志】模块)，以供排查问题。
  6. 例：配置了url为【/test】接口, 那么请求ip:port/vmock/test 即可请求成功，/vmock为默认自定义请求的根目录，一定要携带。
接口日志

该模块有两种进入方式

  1. 由【接口一览】模块点击URL后方的【日志】
  2. 直接左侧菜单
第一种展示的是指定URL的日志，第二种是全部的，并无大区别，从该模块可以看出详细的请求内容和响应内容。
